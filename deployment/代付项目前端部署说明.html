部署结构

整个项目是使用docker-compose部署的
compose中包含3个服务: 前置nginx, redis, 代付项目前端服务

前置条件

  • 服务器ssh地址及密码
  • 服务器前端项目文件夹
  • 应用: docker + docker-compose
  • 代付项目git地址: git@gitlab.axinfu.com:platform/front-end/axinfu-batch-payment-system.git
  • 项目分支: online
  • 代付后台地址
  • 代付后台pramary key
  • 代付后台api key

项目文件准备

1. 拉取代付项目

项目git地址及分支见前置条件

2. 修改项目配置

文件: 项目目录/axinfu-batch-payment-system/src/server/config/api-server.conf.js
修改online配置:

  • serverHost: 代付后台地址的IP或域名
  • serverPort: 代付后台地址的端口
  • pramary_key: 代付后台pramary key
  • X_api_key: 代付后台api key

部署

1. 使用fabric将项目文件同步到服务器

cd到项目fabric目录
执行: fab e:online rsync

期间会要求输入一次登录密码

2. 使用fabric远程执行编译镜像命令

cd到项目fabric目录
执行: fab e:online build

3. 使用fabric远程执行启动服务命令

cd到项目fabric目录
执行: fab e:online server_up

验证

访问: http://[服务器IP]:8096/home/
如果正常打开页面则表示前端项目已经启动
随便输入用户名及密码, 如果提示用户名及密码错, 则表示已经成功连接到后端服务
